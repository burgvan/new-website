Web Experience Toolkit Redesign

Purpose

While working at Treasury Board of Canada Secretariat during and after my co-op, this was one of the projects where my team re-designed key landing pages on the Web Experience Toolkit (WET) website.

Results

We made a prototype of the redesigned version of key landing pages and information architecture with some limitations and delays. We made sure the website still followed web standards and accessibility guidelines.

The Web Experience Toolkit website promotes an open source web framework for public servants and the general population to use, similar to Bootstrap. It's considered to be usable and accessible which is why Government of Canada is now using this framework for Canada.ca. Some of my responsibilities in the team included UX analysis of the current flow for main user groups and to actively use Github for the re-design and gather feedback from the WET community. I used an iterative approach to my design where I developed and refined the design based on feedback and evaluation.

Two rounds of usability testing

My team and I also conducted two rounds of usability testings, prior to the redesign and afterwards for further validation of the new design, as well as tree testing the previous information architecture. We tested users in suitable occupations in the first round from different government departments that are using the open source framework WET. The second round of usability testing was with mainly web publishers and developers that attended a government web conference.

It was clear from the second round of testing that it became easier to find benefits of using WET and the "Getting Started" page became findable and useful. It was also discovered that "Guides", which is a new concept to add to the architecture for web publishers and eComms to read and understand general web and accessibility guides, was clicked on by most users when asked where they would go to check examples of how WET is used rather than navigating to "Implement".
